Store / Server Setup
Run two containers: the store API (paid endpoint) and the store web/proxy (demo UI + /api proxy).
  • Store API image: ghcr.io/tzapac/tzapac-x402-permit2-store-api:latest
  • Store web image: ghcr.io/tzapac/tzapac-x402-permit2-store-web:latest
  • Create a .env file with your store wallet and facilitator URL.
  • Create a Docker network and run both containers.
# .env
FACILITATOR_URL=http://localhost:9090
RPC_URL=https://rpc.bubbletez.com
SERVER_WALLET=0xYOUR_STORE_WALLET
STORE_PRIVATE_KEY=0xYOUR_STORE_PRIVATE_KEY

docker network create x402-store || true

docker run -d --name x402-store-api \
  --network x402-store \
  --env-file .env \
  -p 8001:8001 \
  ghcr.io/tzapac/tzapac-x402-permit2-store-api:latest

docker run -d --name x402-store-web \
  --network x402-store \
  -p 9091:80 \
  ghcr.io/tzapac/tzapac-x402-permit2-store-web:latest
For local dev, you can also run this stack with docker-compose.wallet-poc.yml.
Facilitator Setup
Run the Permit2 facilitator that settles payments on-chain.
  • Facilitator image: ghcr.io/tzapac/tzapac-x402-permit2-facilitator:latest
  • Create a config file and provide a signer private key.
  • Run the container and expose port 9090.
# .env
FACILITATOR_PRIVATE_KEY=0xYOUR_FACILITATOR_PRIVATE_KEY

// bbt_config.json
{
  "port": 9090,
  "host": "0.0.0.0",
  "chains": {
    "eip155:42793": {
      "eip1559": true,
      "signers": ["$FACILITATOR_PRIVATE_KEY"],
      "rpc": [
        { "http": "https://rpc.bubbletez.com", "rate_limit": 100 }
      ]
    }
  },
  "schemes": [
    { "id": "v2-eip155-exact", "chains": "eip155:42793", "enabled": true }
  ]
}

docker run --env-file .env -p 9090:9090 \
  -v $(pwd)/bbt_config.json:/app/bbt_config.json \
  ghcr.io/tzapac/tzapac-x402-permit2-facilitator:latest --config /app/bbt_config.json
For local dev, run the Rust binary with the same config.
